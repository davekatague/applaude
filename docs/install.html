The test

Add a reminder by voice. See it on every device.

You don't need to install anything for this. You're confirming the foundation works.

  1. Open the Claude app on your iPhone. Tap voice mode.
  2. Say: "Add a reminder to buy milk."
  3. Open the Reminders app on the same phone. The reminder should be there.
  4. Open Reminders on your Mac or iPad. It should appear within a minute.

Tip: Applaudé will organise your captures into lists called Knowledge 🪎, Ideas 💡, Entities 🧱, and a few others. The lists already exist in Reminders — you'll see them populate as you use it.

If it didn't sync Settings → your name → iCloud → make sure Reminders is on, on every device. Most "Applaudé isn't working" problems are actually iCloud not turned on somewhere.

№ 01 · Mobile

Already on your phone.

The Claude app on iPhone and iPad has three modes — voice, chat, and projects. All three talk to Reminders. Nothing to install.

Voice. Tap the voice button. Speak. Reminders just works — Claude on iOS uses Apple's own EventKit, the same thing Siri uses. No connector to turn on.

Chat. Turn voice off and you can type, paste, and upload files. Switch voice back on and the conversation continues with the file still in context.

Projects. Inside a project on mobile, you can upload knowledge files once and every voice or chat conversation in that project can use them. This is where the system gets powerful.

№ 02 · Browser

Claude.ai. One click to connect.

Same Reminders, now in your browser. Five minutes. The connector is hosted by Anthropic, so there's nothing to install on your computer.

This is the workhorse surface. Use it for longer thinking, file uploads, planning, and anything that benefits from a wider screen than your phone.

Turn on the connector

  1. Go to claude.ai and sign in.
  2. Open your profile menu → Connectors.
  3. Find Apple Reminders in the list and click Connect.
  4. Approve the permission prompt. Allow access to your Reminders.
  5. Open a new chat. Type: "List my Reminders lists." You should see them.

Install the Applaudé skill

The skill teaches Claude how to structure information — so a person you mention becomes a linked entity, not just plain text. Five steps.

  1. In Claude.ai, create a new Project. Call it Applaudé.
  2. Upload SKILL.md from the repo to project knowledge.
  3. Paste the project instructions from bridges/claudeai.md.
  4. Start a new chat inside that project.
    5. A small elegance Your Reminders lists are named in plain English (Knowledge, Entities, Projects) so they work across every Claude surface. When Claude talks about them, you'll see the decorated versions — Knowledge 🪎, Entities 🧱, Projects 🚧. Warmth in the experience, portability in the storage. Nobody notices the seam.
№ 03 · Desktop

Claude on your Mac.

The Mac app needs a small install — about fifteen minutes if you're comfortable in Terminal. Same setup also works for Cowork.

Before you start The desktop install needs Terminal — the command-line app on your Mac. If "Terminal" makes you nervous, just stay on Claude.ai. You'll get most of the way there without it.
Option A

Stay on browser

Skip this section. Claude.ai gives you 90% of the experience with zero Terminal time.

Option B

Push through

Follow the steps below. You'll unlock Desktop, Cowork, and the foundation for Claude Code.

The Mac app uses a community-built bridge called mcp-server-apple-events, made by a developer named FradSer. It's the same code that Cowork and Claude Code use, so one install covers all three. It's open source.

Install the bridge

  1. Install Claude Desktop from claude.ai/download.
  2. Open Terminal (Applications → Utilities → Terminal).
  3. Find your configuration file: 
    open ~/Library/Application\ Support/Claude/
  4. Open claude_desktop_config.json in TextEdit (or any text editor). If the file doesn't exist, create it.
  5. Paste this in (merge it with anything that's already there): 
    {
  "mcpServers": {
    "apple-reminders": {
      "command": "npx",
      "args": ["-y", "mcp-server-apple-events"]
    }
  }
}
  6. Save the file. Quit Claude Desktop completely. Reopen it.
  7. The first time Claude tries to read Reminders, macOS will pop up a permission dialog. Click OK.
  8. Test: ask "List my Reminders lists."

If something goes wrong Open System Settings → Privacy & Security → Reminders and make sure Claude is in the list with the toggle on. About nine out of ten "it doesn't work" problems are a missed permission.

Cowork

If you have Cowork, the setup is identical — Cowork reads the same config file. Once Desktop is working, Cowork works too.

№ 04 · Claude Code

For developers.

Claude Code is for people who build with code. It can read and write files on your Mac, run shell commands, and do work the other surfaces can't.

If you're not a developer You don't need this. The first three surfaces cover everything most people need. Skip ahead.

Most of the install is already done — Claude Code uses the same MCP bridge you installed for Desktop. You just point it at the Applaudé skill folder.

Install

  1. Install Claude Code: 
    npm install -g @anthropic-ai/claude-code
  2. Clone the Applaudé skill into your Claude Code skills folder: 
    git clone https://github.com/davekatague/applaude.git \
  ~/.claude/skills/applaude
  3. The Apple Events MCP is already available from your Desktop install. No extra config needed.
  4. Start Claude Code and try: "list my Reminders lists" or "capture this idea".

Where this came from

Standing on shoulders.

Applaudé wouldn't exist without two people who got there first.

Andrej Karpathy built a personal wiki that showed what it looks like to organise knowledge for reading. Click a link, follow a thought, learn something. Beautiful and clean.

Arscontexta took the same idea and asked: what if knowledge was organised for an agent to think with? Structured, linked, machine-readable.

Applaudé is the third step: knowledge organised so you can use it in real life — mid-drive, mid-call, mid-decision. By voice. On the devices you already own.
